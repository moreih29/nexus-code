# external-codex

스냅샷 날짜: 2026-04

---

## 개요

OpenAI가 공식 배포하는 CLI 기반 AI 코딩 하네스. 비공개 소스. 세션 스키마 변동이 세 하네스 중 가장 잦아 어댑터 지속 보수 부담이 높음.

---

## 배포 방식

- **런타임**: Node.js 기반 CLI. npm 또는 npx를 통해 설치.
- **배포 경로**: npm 레지스트리 공식 패키지명 `@openai/codex`. OpenAI 공식 배포.
- **플랫폼**: macOS / Linux / Windows (Node.js 지원 범위).

---

## 세션 저장 경로 및 포맷

- **포맷**: JSON 또는 JSONL 세션 파일. 버전에 따라 포맷이 달라질 수 있음.
- **경로**: 사용자 홈 디렉토리 또는 프로젝트 로컬 디렉토리 내 codex 전용 경로. 정확한 경로는 버전·설정에 따라 다름.
- **주의**: 세션 파일 포맷 및 경로가 2026-04 현재 자주 변경됨. 특정 버전에서 확인한 경로·포맷이 다음 릴리스에서 변경될 수 있음.

---

## 관찰 가능한 이벤트

**JSON 출력**:
- `--json` 또는 유사 플래그로 구조화된 JSON을 stdout으로 출력하는 모드 지원.
- 출력 스키마(이벤트 타입, 필드명, 중첩 구조)가 릴리스마다 변동됨 — 파서 방어 코드 필수.

**세션 파일**:
- 세션 파일을 fsnotify로 감시해 신규 내용을 읽는 방식 사용 가능.
- 파일 포맷 및 이벤트 구조 변동 빈도가 높아, 어댑터 측에서 버전별 파싱 분기 또는 관대한 파싱(unknown 필드 무시) 전략이 필요.

**nexus-code 통합 지침**:
- 외부 의존을 최소로 설계. 스키마가 바뀌어도 핵심 기능(완료 여부, 도구 호출 감지, 에러)만 추출하는 방어적 어댑터 구현 권장.
- 세부 필드 파싱에 의존하는 기능은 codex 어댑터에서 제한.

---

## per-turn spawn + `--resume` 지원

- **per-turn spawn**: 2026-04 기준 지원 여부 및 동작 방식이 버전마다 달라지는 상태. 안정적 동작을 가정할 수 없음.
- **`--resume <sessionId>`**: 2026-04 기준 세부 플래그명·동작 스펙이 자주 바뀜. 공식 문서 기준으로도 릴리스 간 불일치가 보고됨.
- nexus-code 통합 전략: 세부 포맷·플래그에 최소한으로 의존하는 어댑터 인터페이스 설계. 변경 발생 시 어댑터 레이어만 수정.

---

## 알려진 IME / 터미널 / CJK 관련 이슈

- 2026-04 기준 공식 이슈 트래커에서 CJK/IME 관련 번호가 지정된 미해결 이슈는 확인되지 않음.
- CLI 특성상 IME 처리는 호스트 터미널 에뮬레이터에 의존.
- nexus-code에서 xterm.js 내 실행 시 IME 이슈는 xterm.js 레이어에서 관리.

## 스키마 안정도

- **세션 파일 스키마**: 세 하네스 중 변동이 가장 잦음. OpenAI 릴리스 주기에 따라 포맷이 공지 없이 변경되는 사례가 있음.
- **JSON 출력 스키마**: 마찬가지로 릴리스 간 필드명·구조 변동 빈도 높음.
- **변동 빈도 평가**: 높음. claude-code(중간), opencode(낮음)와 비교해 가장 잦은 변동.
- **어댑터 보수 부담**: 높음. nexus-code MVP에 codex 어댑터를 포함하는 만큼, Issue 2 일정 산정에 어댑터 업데이트 비용을 명시적으로 반영.
