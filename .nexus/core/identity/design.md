# Design

## UI 철학: 집중과 전환
- 현재 맥락에 필요한 정보만 보여주고, 빠르게 전환한다
- 에이전트 선택 → 해당 컨텍스트 자동 표시
- 불필요한 정보는 숨기되, 필요할 때 드러내는 구조

## 다중 워크스페이스
- 하나의 앱에서 여러 워크스페이스를 관리
- 각 워크스페이스는 독립된 에이전트 그룹과 상태를 가짐
- 워크스페이스 간 빠른 전환 (구체적 UI 패턴은 미정)
- 다른 워크스페이스에 집중 중이어도 앱 내 알림으로 승인 요청·에러를 인지 가능

## 아키텍처: 균형
- 모듈성: 기능 단위로 분리되어 독립적으로 개발·테스트 가능
- 단순함: 각 모듈의 책임이 명확하고, 모듈 간 결합도 최소화
- 성능: 리소스 사용량과 응답성 의식
- 신뢰성: 에러 처리와 상태 관리에 일관된 패턴

## 기술 스택
- 미정 — 설계 완료 후 프레임워크 결정
