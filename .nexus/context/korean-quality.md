# Korean Quality

## 지원 범위

| 언어 | 입력·렌더링·경로 처리 | UI 번역 |
|---|---|---|
| 영어 | 공식 보장 | 공식 언어 (단일) |
| 한국어 | 공식 보장 | 제공하지 않음 |
| 일본어 | best-effort | 제공하지 않음 |
| 중국어 | best-effort | 제공하지 않음 |

UI는 영어 단일이다. 한국어 UI 번역은 초기 단계에서 제공하지 않으며, 커뮤니티 기여는 허용한다.

---

## 한국어 리그레션 분류 정책

한국어 입력·렌더링·경로 처리에서 발생한 리그레션은 **블록커 이슈**로 분류한다.

- 블록커로 분류된 이슈는 해당 릴리스 전에 반드시 수정되어야 한다.
- 일본어·중국어 리그레션은 블록커가 아니다. 수정을 권장하지만 릴리스를 막지 않는다.

---

## 릴리스 전 필수 통과 체크리스트 (한국어 IME·렌더링)

다음 7개 항목을 모두 통과해야 릴리스할 수 있다.

1. 한글 IME 조합 중 커서가 입력 위치에 정확히 표시된다.
2. 조합 중 Enter는 조합 확정만 수행하며, 두 번째 Enter가 submit으로 동작한다.
3. 한글 double-width가 정확히 계산된다 (`ㅎ`, `가`, 한글 일반 포함).
4. 파일명과 경로가 NFC로 정규화된다 (macOS NFD → NFC).
5. 조합 중 문자 소실이 없다.
6. 한글 입력 lag가 16ms 미만이다 (평균 키 입력 기준).
7. 기본 폰트 폴백 스택에 Noto Sans KR과 D2Coding이 포함된다.

## 회피 대상 업스트림 이슈

다음 이슈들은 사전에 인지하고 회피 전략을 구현해야 한다.

**xterm.js IME 조합 위치 이슈 (#5734)**
IME 조합 중 조합창(composition window)이 입력 위치가 아닌 엉뚱한 곳에 표시되는 버그다.
체크리스트 1번 항목과 직결된다.
회피 수단은 `composingstart` / `compositionend` 이벤트 기반 오버레이 패치이며, 필요 시 fork 또는 업스트림 PR로 대응한다.

**Enter 두 번 문제 (Cursor·Warp 사례)**
IME 조합 중 Enter를 누르면 조합 확정과 submit이 동시에 일어나는 동작이다.
일부 터미널 에뮬레이터와 AI 코딩 도구에서 재현된 알려진 결함이다.
체크리스트 2번 항목에서 명시적으로 검증한다.

**Claude Code CLI 미해결 이슈 (#16372, #22732)**
Claude Code CLI에서 IME 조합 문자 소실과 커서 위치 불일치가 보고된 상태다.
체크리스트 5번 항목이 이 동작을 직접 검증한다.
우리 터미널 레이어에서 완충할 수 있는 부분은 완충하고, 업스트림 수정 여부를 지속 모니터링한다.
