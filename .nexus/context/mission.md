## 프로그램 가치
- 하나의 윈도우에서 멀티 워크스페이스를 다룸
- 각 워크스페이스는 각자의 작업 공간을 가짐
- vscode에서 제공하는 ide 기능과 vscode에서 제공하지 못하는 멀티 워크스페이스 기능을 합친 것

## 개발 원칙
- 하나의 파일에 너무 많은 기능을 담지 않음
  - 함수, 클래스 또한 하나에 너무 많은 기능을 담지 않음
- 책임 분리 원칙에 따라 기능을 개발하며, 모듈화 지향
- 폴더 구조는 사람이 인지하기 쉽도록 계층화함
- 클래스 및 함수에는 주석을 통해 의미를 반드시 작성 (단, M0, Phase1 같은 일시적인 내용은 담지 않음)
- 테스트는 이 프로그램에 정말 필요한 시나리오 기반의 테스트만 진행함
  - 라이브러리 / 타입 시스템 / 스키마가 이미 보장하는 동작은 다시 검증하지 않음 (zod 자체 검증, 타입 재진술, mock 인자 echo 류는 시나리오가 아님)
  - 상위 시나리오 테스트가 이미 커버하는 부분집합은 별도 케이스로 두지 않음
