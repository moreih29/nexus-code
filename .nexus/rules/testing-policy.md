# 테스트 정책

## 우선순위 체계

| 단계 | 범위 | 예시 |
|------|------|------|
| **P0** | 핵심 경로 — 실패 시 앱이 무너짐 | 워크스페이스 생성/삭제, 세션 시작/종료, 권한 훅, 설정 저장/조회 |
| **P1** | 주요 기능 — 사용자 경험에 직접 영향 | 세션 복원, SSE 이벤트 전송, 설정 동기화, 파일 트리/Git 통합 |
| **P2** | 부가 기능 — 품질과 디테일 | UI 상호작용, 엣지 케이스, 시각적 세부사항 |

## 도구

- **Unit**: Vitest — `packages/server`, `packages/web` 각 패키지에서 독립 실행
- **E2E**: Playwright — `bun run test:e2e` (반드시 `bun run build` 선행)

## 규칙

- 새 기능에는 최소 P0 또는 P1 테스트를 동반할 것
- 버그 수정은 재발 방지 회귀 테스트 추가를 원칙으로 함
- **권한 제어·세션 생명주기·설정 동기화** 관련 수정은 E2E 검증 필수
- 서버/웹 각 패키지의 테스트는 상호 독립적이어야 함 (교차 의존 금지)
- 테스트 추가 커밋은 `test:` 프리픽스를 사용하고 단계(P0/P1/P2)를 제목에 명시
