# 커밋 메시지 규약

## 형식

```
<type>: <한글 제목> [— <부제목>]
```

## 허용 type

- `feat` — 새 기능
- `fix` — 버그 수정
- `refactor` — 동작 변화 없는 구조 개선
- `test` — 테스트 추가/변경
- `docs` — 문서(코드 변경 없음)
- `chore` — 빌드/설정/잡무
- `merge` — 브랜치 머지 (`merge: <branch> — <설명>`)

## 규칙

- 제목은 한글로. 명사형 또는 동사형 통일
- 상세 설명이 필요할 때는 em dash(`—`)로 부제목 연결
- 하나의 커밋은 하나의 관심사만 담을 것
- 브랜치 네이밍: `fix/*`, `feat/*`, `chore/*`, `refactor/*`

## 예시

- `feat: 워크스페이스별 디버그 로깅 시스템 + 클라이언트 디버그 로그 정리`
- `fix: 권한 제어 아키텍처 정상화 — hook 재주입, SSE push, tool_result 파싱`
- `refactor: 서버/웹 아키텍처 리팩토링 — 파편화 해소, 중복 제거, 모듈화`
- `test: P0 테스트 추가 — 서버 19건 + 웹 42건`
- `merge: fix/settings-sync-bugs — 설정 동기화 버그 수정 + 아키텍처 정상화 + 테스트 강화`
